# The Self-Pruning Neural Network

This repository contains the solution for the "Self-Pruning Neural Network" assessment task. The goal is to design and implement a neural network that learns to prune itself during the training process by identifying and dynamically removing its own weakest connections.

## Architecture & Implementation

The solution implements a custom `PrunableLinear` layer in PyTorch. 
Instead of the standard `torch.nn.Linear` layer, this custom layer maintains an additional learnable parameter tensor called `gate_scores` of the same shape as the weight tensor.

During the forward pass:
1. The `gate_scores` are passed through a Sigmoid function to squash the values strictly between 0 and 1.
2. The regular expected weights are multiplied element-wise by these gate values (`pruned_weights = self.weight * gates`).
3. The standard linear operation is then performed using the pruned weights.

### Why an L1 Penalty on Sigmoid Gates Encourages Sparsity

If the network is trained using only the standard classification loss (Cross-Entropy), all gates would remain near their initial states (or grow to 1) because the network has no incentive to remove connections. 

To encourage pruning, an L1 regularization term (the sum of the absolute values of the gates) is added to the total loss:
`Total Loss = ClassificationLoss + λ * SparsityLoss`

Because the gates are the output of a Sigmoid function, their values are positive, meaning the L1 norm is simply the sum of all gate values across the network. The L1 penalty applies a constant, linear downward force pushing the gate values towards zero. When the negative gradient from the sparsity penalty outweighs the positive gradient from the classification loss for a specific connection, the gate score is pushed negative, driving the Sigmoid output to exactly 0. This effectively "prunes" that weight from the network.

*Note: To ensure the gates can respond effectively to the sparsity penalty in a reasonable number of epochs, the optimizer relies on separate parameter groups. It applies a much higher learning rate specifically to the `gate_scores` parameters while keeping the standard learning rate for the structural weights.*

## Results: Sparsity vs. Accuracy Trade-off

The network consists of a standard CNN feature extractor followed by two `PrunableLinear` classifier layers. It was trained on the CIFAR-10 dataset for 20 epochs across five different levels of sparsity penalty ($\lambda$).

A gate is considered "pruned" if its final sigmoid value falls below a strict threshold of `0.01`.

| Lambda Level | Penalty (λ) | Test Accuracy | Sparsity Level (%) |
| :--- | :--- | :--- | :--- |
| None | 0.0 | 81.47% | 1.77% |
| Low | 1e-4 | 81.58% | 88.83% |
| Medium | 1e-3 | 81.20% | 96.24% |
| High | 1e-2 | 72.06% | 99.61% |
| Very High | 5e-2 | 35.15% | 99.98% |

### Analysis

1. **Over-parameterization (The Free Lunch):** The "Low" ($\lambda$=1e-4) experiment highlights that the dense layers were massively over-parameterized. The network successfully discarded **nearly 89% of its connections** while actually ***increasing* its accuracy slightly** (from 81.47% to 81.58%). This demonstrates that the pruning mechanism also acted as an effective regularizer against overfitting.
2. **The Sweet Spot:** The "Medium" ($\lambda$=1e-3) experiment achieved an **incredible 96.24% sparsity**. This means the model retained only ~3.7% of its classifier weights, yet suffered a **negligible accuracy drop of 0.27%** compared to the baseline.
3. **The Trade-off Limit:** Pushing $\lambda$ to 1e-2 and 5e-2 forced the network to aggressively prune highly important structural weights (**reaching >99% sparsity**). As the network was starved of connections, this predictably led to a **sharp degradation** in classification accuracy.

## Gate Value Distribution

*(Please reference the `gate_distribution.png` generated by the script).*

The output plot shows histograms of the final sigmoid gate values for each $\lambda$ run. For runs with active pruning ($\lambda > 0$), there is a massive spike exactly at `0.0` (representing the pruned, dead connections), and a much smaller secondary cluster closer to `1.0` (representing the small fraction of surviving, highly active connections). The baseline run (where $\lambda=0$) shows a smooth distribution hovering around the 0.5 initialization point.

## How to Run

1. Ensure PyTorch and TorchVision are installed in your environment.
2. Execute the main framework:
   ```bash
   python self_pruning.py
   ```
   
The script will download CIFAR-10 into a local `/data` directory automatically, run the training pipeline across all predefined $\lambda$ values, and print real-time training progress to the console.

Upon completion, it dynamically generates:
- `results.json`: A data export of the metrics.
- `gate_distribution.png`: The visual histogram analysis of the final gate values.
